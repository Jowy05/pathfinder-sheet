## 6. Notes / Risks

1. **Buffs/Adjustments scope.** The existing `#conditions-panel` contains the conditions grid AND the Buffs / Campaign-Adjustments sub-sections (L446–454). Those MUST remain visible even when the conditions list is collapsed. The HTML replacement in §2 closes the `collapsible-body` right after the `cond-grid` — the remaining markup (`<div class="divider"></div>` … through `<button class="btn-add" id="add-tweak-btn" …>`) stays below it as a sibling inside `#conditions-panel`. **Implementation note:** after the edit, `#conditions-panel` will have TWO children: `<h2 class="collapsible-header">` + `<div class="collapsible-body">` (with the cond grid), then the original divider + buffs + adjustments siblings. Counting existing closing `</div>`s: the original `</div>` on L445 that closed `.cond-grid` is now replaced by `</div><!-- /collapsible-body -->` which closes `#conditions-body`. The buffs/adjustments block is naturally OUTSIDE the new body. No stray tags.

2. **Existing `cond-stunned` wiring is kept intact.** `recalc()` at L4115 & L4140 reads `cond-stunned.checked` as a "no-DEX, –2 AC" flag. Semantically that IS the *stunned* condition (the mislabel was only in the visible text). With the new i18n re-pointed to `cnd.stunned`, behavior is unchanged; we just add a separate `cond-dazed` row whose penalties (no actions, keeps AC) are not currently applied and will need a follow-up patch to integrate into `recalc()`.

3. **`flat-footed` id.** Hyphen in `cond-flat-footed` is intentional (kebab-case matches other kebab ids such as `cond-prone`). The data-cond attribute uses `flatfooted` (no hyphen) for cleaner key lookup; the map `PF1E_COND_CB_ID.flatfooted = 'cond-flat-footed'` bridges them. Note PF1e already has a `#ac-flat-footed` readout — no collision: that one is `ac-flat-footed`, ours is `cond-flat-footed`.

4. **No new `.recalc` on timer inputs.** Round-number fields do NOT carry class `recalc`, so typing in them won't trigger the global recalc listener (L3688). The summary redraw is handled by a panel-scoped `input` listener in `initConditionsTimer()`.

5. **Programmatic `checked = false` does not fire `change`.** All three code paths that uncheck a condition programmatically (chip click, timer-expiry in `advanceConditionsRound`, `clearAllConditions`, `applyCharacter`) explicitly call `recalc()` so the prone / stunned / exhausted stat adjustments are re-evaluated immediately.

6. **Save-file backwards compatibility.** Old exports lack a `conditions` key; `applyCharacter()` / `importData()` fall back to the per-id generic-input restore loop which already handles `cond-*` checkbox values. Round-timer fields in old saves will simply be absent → empty (= permanent), which is the safest default. New exports carry both formats (per-id booleans AND the `conditions` object) so downgrading doesn't corrupt anything either.

7. **`collectCharacter`/`applyCharacter` naming.** The task spec asks for these exact names. They are now the canonical entry points; `exportData`/`importData` continue to exist and `exportData` now embeds the structured `conditions` block for legibility in the downloaded JSON. Future patches should prefer `collectCharacter`/`applyCharacter`.

8. **Collapsible header registration.** `setupListeners()` uses a hard-coded array of header ids (L3762). We deliberately do NOT add `conditions-header` to that array — `initConditionsTimer()` wires its own collapsible so it can also render the summary on first paint in one place.

9. **Mobile/Responsive.** `.cond-grid` already responds down to 1-col < 480px (style.css L798–804). The new `.cond-rounds-input` is 38px wide and right-aligned inside `.cond-item` via `margin-left:auto`; it fits the existing 2-col layout without overflow. On very narrow screens the timer input sits below the label because `.cond-item` is `display:flex; align-items:flex-start`.

10. **Print styles.** Per §1 CSS, the `@media print` block hides `.cond-next-round-btn`, `.cond-clear-btn`, and `.cond-rounds-input`. The summary chip bar is kept visible so the printed sheet still shows active conditions.

11. **Accessibility.** Collapsible header gets `role="button"`, `tabindex="0"`, `aria-expanded`, `aria-controls`. Rounds input has `min="0"` and a `placeholder="∞"` that becomes visible only when empty (= permanent). Chips have a `title` attribute explaining the click-to-remove affordance. i18n covers all new visible strings.

12. **Extending to auto-apply new-condition penalties.** Intentionally out-of-scope for P-3. Follow-up P-4 could wire `bleed` into HP regen, `pinned`/`helpless` into AC/DEX, and `dazed`/`stunned` into action economy (display-only or forced recalc).
