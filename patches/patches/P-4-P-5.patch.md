# P-4 · Validador de prerequisitos · Patch Spec

## 0. feats.json prerequisite format (examples from actual file)

Structure per entry (free-text, comma-separated, ES/EN localised fields):

```json
{ "id": "ataque_poderoso",    "name": "Ataque Poderoso",          "name_en": "Power Attack",
  "prerequisites":    "FUE 13, BAB +1",
  "prerequisites_en": "STR 13, BAB +1" }
```

Other representative shapes found in `feats.json`:

```json
{ "id": "hendedura",           "prerequisites": "FUE 13, Ataque Poderoso, BAB +1" }
{ "id": "gran_hendedura",      "prerequisites": "FUE 13, Hendedura, Ataque Poderoso, BAB +4" }
{ "id": "soltura_mayor_arma",  "prerequisites": "Soltura con el arma, BAB +1, Guerrero nivel 8" }
{ "id": "canalizacion_rapida", "prerequisites": "Saber (religión) 5, Canalizar Energía" }
{ "id": "canalizacion_selectiva","prerequisites": "CAR 13, Canalizar Energía" }
{ "id": "disparo_bocajarro",   "prerequisites": "Ninguno" }
{ "id": "apresurar_conjuro",   "prerequisites": "—" }
{ "id": "critico_sangrante",   "prerequisites": "Crítico Mejorado, BAB alto" }
{ "id": "estilo_grulla",       "prerequisites": "Esquiva, Golpe sin Armas Mejorado, BAB +2 o Monje 1" }
```

Observations that drive parser design:
- Attribute minimums: Spanish `FUE|DES|CON|INT|SAB|CAR` + number, English `STR|DEX|CON|INT|WIS|CHA` + number.
- BAB: `BAB +N` (numeric). Free-text fallbacks like `BAB alto`, `BAB elevado` → unparseable → neutral `❓`.
- Class level: `Guerrero nivel 8`, `Monje 1`, `Fighter level 8`, `Wizard level 1`, `Mago nivel 1`, `Clérigo Neutral`.
- Caster level: `Nivel de lanzador N` / `Caster level N`.
- Skill ranks: `Saber (religión) 5`, `Knowledge (religion) 5`, `Montar 1 rango`, `Ride 1 rank`, `Acrobacias`.
- Feat references: the feat name itself (e.g. `Ataque Poderoso`, `Esquiva`, `Pericia en Combate`). Resolve by searching `DB.feats` for a feat whose ES/EN `name`/`name_en` appears in the token.
- `Ninguno` / `None` / `—` → always met.
- "or" clauses (`BAB +2 o Monje 1`, `BAB +2 or Monk 1`) → met if either side holds.
- Free text (e.g. `Aptitud para lanzar conjuros arcanos`, `Competencia con el arma`, `rasgo Resistente`, `Aliado en formación`) → unparseable → `❓` with the raw token shown.

// --- Token parsing ----------------------------------------------------
// returns { kind:'attr|bab|class|caster|skill|feat|none|unknown', ok, label, reason }
function parseFeatPrereqToken(raw, ctx){
  const orig = raw.trim();
  const tk = _normFeatTxt(orig);
  if(!tk || tk==='-' || tk==='—' || tk==='ninguno' || tk==='none' || tk==='n/a')
    return { kind:'none', ok:true, label:orig };

  // --- OR branches ("A o B", "A or B") ---
  const orParts = tk.split(/\s+(?:o|or)\s+/);
  if(orParts.length>1){
    const subs = orParts.map(p=>parseFeatPrereqToken(p,ctx));
    const anyOk = subs.some(s=>s.ok);
    const anyUnknown = subs.some(s=>s.kind==='unknown');
    return {
      kind:'compound',
      ok: anyOk || (!subs.some(s=>s.kind!=='unknown') && anyUnknown),
      label: orig,
      reason: anyOk ? null : subs.map(s=>s.label).join(' / ')
    };
  }

  // --- Attribute minimum: "fue 13", "str 13" ---
  let m = tk.match(/^(fue|des|con|int|sab|car|str|dex|wis|cha)\s+(\d+)$/);
  if(m){
    const a = FEAT_ATTR_ALIAS[m[1]]; const need = parseInt(m[2]);
    const have = ctx.attr[a] ?? 10;
    return { kind:'attr', ok: have>=need, label: orig,
             reason: `${m[1].toUpperCase()} ${have}/${need}` };
  }

  // --- BAB ---
  m = tk.match(/^bab\s*\+?\s*(\d+)/);
  if(m){
    const need=parseInt(m[1]);
    return { kind:'bab', ok: ctx.bab>=need, label: orig,
             reason:`BAB +${ctx.bab}/+${need}` };
  }
  if(/^bab\b/.test(tk)) return { kind:'unknown', ok:false, label:orig }; // "BAB alto"

  // --- Caster level ---
  m = tk.match(/^(?:nivel\s+de\s+lanzador|caster\s+level)\s+(\d+)/);
  if(m){
    const need=parseInt(m[1]);
    return { kind:'caster', ok: ctx.casterLvl>=need, label:orig,
             reason:`CL ${ctx.casterLvl}/${need}` };
  }

  // --- Class level: "guerrero nivel 8", "fighter level 8", "monje 1", "monk 1"
  m = tk.match(/^([a-záéíóúñ]+)\s+(?:nivel|level)\s+(\d+)/)
   || tk.match(/^([a-záéíóúñ]+)\s+(\d+)$/);
  if(m){
    const key = FEAT_CLASS_ALIAS_ES[m[1]] || FEAT_CLASS_ALIAS_EN[m[1]];
    const need=parseInt(m[2]);
    if(key){
      const have = ctx.classLevels[key]||0;
      return { kind:'class', ok: have>=need, label:orig,
               reason:`${m[1]} ${have}/${need}` };
    }
  }

  // --- Skill rank: "montar 1 rango", "ride 1 rank", "saber (religion) 5"
  m = tk.match(/^(.+?)\s+(\d+)\s*(?:rango|rank)s?$/)
   || tk.match(/^(.+?)\s+(\d+)$/);
  if(m){
    const skKey = FEAT_SKILL_ALIAS[m[1].trim()];
    if(skKey){
      const need=parseInt(m[2]);
      const have=ctx.skillRanks[skKey]||0;
      return { kind:'skill', ok: have>=need, label:orig,
               reason:`${m[1]} ${have}/${need}` };
    }
  }

  // --- Feat reference: match against DB.feats names (ES + EN) ---
  const feats = DB.feats || [];
  const hit = feats.find(f =>
    _normFeatTxt(f.name)===tk || _normFeatTxt(f.name_en)===tk
  );
  if(hit){
    const want = _normFeatTxt(tData(hit,'name'));
    const wantEn = _normFeatTxt(hit.name_en);
    const ok = ctx.ownFeats.has(want) || ctx.ownFeats.has(wantEn);
    return { kind:'feat', ok, label: orig };
  }

  return { kind:'unknown', ok:false, label:orig };
}

function parseFeatPrereqs(rawEs, rawEn, ctx){
  const raw = (LANG==='en' && rawEn) ? rawEn : (rawEs || rawEn || '');
  if(!raw) return { status:'none', missing:[], unknown:[] };
  const norm = _normFeatTxt(raw);
  if(!norm || norm==='-' || norm==='—' || norm==='ninguno' || norm==='none')
    return { status:'ok', missing:[], unknown:[] };
  const tokens = raw.split(/,\s*/).map(t=>t.trim()).filter(Boolean);
  const missing=[], unknown=[];
  tokens.forEach(tok=>{
    const p = parseFeatPrereqToken(tok, ctx);
    if(p.kind==='unknown') unknown.push(p.label);
    else if(!p.ok)         missing.push(p.reason || p.label);
  });
  let status='ok';
  if(missing.length) status='warn';
  else if(unknown.length) status='unknown';
  return { status, missing, unknown };
}

function refreshFeatPrereqChips(){
  if(!DB.feats) return;
  const ctx = getCharPrereqContext();

    const nameInput = card.querySelector('.feat-name');
    if(!nameInput) return;
    const nm = _normFeatTxt(nameInput.value);
    const feat = (DB.feats||[]).find(f =>
      _normFeatTxt(f.name)===nm || _normFeatTxt(f.name_en)===nm
    );
    let chip = card.querySelector('.feat-req-chip');
    if(!chip){
      chip=document.createElement('span');
      chip.className='feat-req-chip';

      card.insertBefore(chip, card.firstChild); // before .feat-dot

      });

      chip.addEventListener('focus',      e=>{

      });
      chip.addEventListener('blur', _hideFeatTip);
    }
    if(!feat){
      chip.className='feat-req-chip unknown';
      chip.textContent='❓';
      chip.dataset.tip = t('feat.reqCustom');

    }

    chip.textContent = res.status==='ok'?'✓':(res.status==='warn'?'⚠️':'❓');
    const parts=[];
    if(res.missing.length)
      parts.push(`<b>${t('feat.reqMissing')}</b><ul>${res.missing.map(x=>`<li>${x}</li>`).join('')}</ul>`);
    if(res.unknown.length)
      parts.push(`<b>${t('feat.reqUnparsed')}</b><ul>${res.unknown.map(x=>`<li>${x}</li>`).join('')}</ul>`);
    if(!parts.length) parts.push(t('feat.reqAllMet'));
    chip.dataset.tip = parts.join('');
  });
}

## 6. Notes

- `DB.feats` is populated at line 2053 (`feats.json`); validation silently no-ops when not yet loaded.
- The chip is purely informational — `addFeat` never blocks. A custom-named feat (not matching any DB entry) gets the neutral `❓` chip.
- Chip is placed **before** `.feat-dot` so print CSS (`.btn-remove { display:none }`) is unaffected; print rule hides its border so it doesn't stand out.
- Attribute totals read from `str-total`/`dex-total` etc., which already reflect racial + enhancement + temporal buffs — matches Pathfinder rules (prereqs check the *current* score, not base).
- Caster level uses `caster-level-total`/`caster-level` if present, otherwise falls back to the highest full-caster class level (approximation; adequate for the common `Nivel de lanzador N` prereq).
- Parser purposefully lenient: if it cannot classify a token, it surfaces `❓` with the raw fragment in the tooltip — user keeps visibility, no false negatives.
- Re-evaluation triggers: (a) every `recalc()` (already wired to `.recalc` inputs, attr edits, class changes), (b) feat add/remove, (c) feat-name input.
- Languages: chip glyphs and tooltip labels localised via `t(...)`; prereq text itself is shown in the active language (`tData` fallback).

## 6. Notes

- **Why not move the existing panel?** It already participates in the saved-notes list (`_NOTE_FIELDS.combat`) and uses a relatively large textarea (`saves-notes-area`). Moving it into `#combat-panel` would either shrink the Combat panel visually or bloat it. A **mini mirror** (3-row compact) sitting next to BAB/HP gives the DM-side affordance while leaving the authoring/history UX untouched.
- The mini textarea has `max-height:3.6em` via CSS to guarantee ≤3 visible rows; content overflowing is still scrollable inside it, and the "Abrir panel completo" button scrolls+focuses the full editor for extended edits.
- The full ↔ mini sync uses a reentrancy guard to avoid input-loop churn and dispatches a bubbling `input` event on the full textarea so any future observers (autosave, typing indicator) keep working.
- Print stylesheet hides the pin (`@media print { #combat-notes-pin{display:none;} }`) since the full notes already print from their canonical panel.
- No change to `collectCharacter()`/`applyCharacter()` is needed — the mini isn't persisted, it just mirrors `#combat-notes`.
