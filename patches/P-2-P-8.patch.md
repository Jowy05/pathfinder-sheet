# P-2 · Descanso personalizado · Patch Spec

Replaces the single "Nuevo Día — Recuperar Conjuros" button with two
Pathfinder-style rest options:

- **Descanso corto (1h)** — refreshes concentration/encounter pools
  (ki, panache, arcane reservoir, bardic performance, bombs,
  inspiration, focus, grit, martial flexibility). Does NOT recover
  spell slots per day or HP.
- **Descanso largo (8h)** — full daily reset: all per-day features
  (including rage / fervor / smite / channel / lay on hands / judgment
  / bane / studied target / bloodrage / wild shape / touch of
  corruption), spell slots, HP to full, temp HP cleared, *fatigued*
  removed, *exhausted* downgraded to *fatigued*.

Both use the existing `_autoKey` identifiers from `CLASS_AUTO_FEATURES`
to know which feature-card belongs to which tracker.

function _resetFeatureCard(card) {
  card.querySelectorAll('.tracker-bubble.used').forEach(b => {
    b.classList.remove('used');
    b.title = t('tt.avail.click');
  });

/* Short rest (1h): refresh concentration/encounter pools only.
   Does NOT touch spell slots, HP or conditions. */
function restShort() {
  document.querySelectorAll('.feature-card[data-auto-key]').forEach(card => {
    const key = card.dataset.autoKey;
    if(REST_RESET_KEY[key] === 'short') _resetFeatureCard(card);
  });
  try { logRoll && logRoll(t('m.restShortDone'), ''); } catch(_){}
}

/* Long rest (8h): full daily reset.
   · All per-day trackers (short + long).
   · Spell slots (delegates to existing newDay()).
   · HP to max, temp HP cleared.
   · fatigued → cleared. exhausted → becomes fatigued. */
function restLong() {
  // 1. All auto-feature trackers (every class tracker)
  document.querySelectorAll('.feature-card').forEach(_resetFeatureCard);
  // 2. Spell slots + expended marks (existing logic)
  newDay();
  // 3. HP restore
  const hpMax = parseInt(gv('hp-max')) || 0;
  if(hpMax > 0) sv('hp-current', hpMax);
  sv('hp-temp', 0);
  try { updateHPBar(); } catch(_){}
  // 4. Fatigue / exhaustion
  const exh = document.getElementById('cond-exhausted');
  const fat = document.getElementById('cond-fatigued');
  if(exh && exh.checked) {
    exh.checked = false;
    if(fat) fat.checked = true;  // exhausted → fatigued after 8h rest
  } else if(fat) {
    fat.checked = false;
  }
  try { recalc(); } catch(_){}
  try { logRoll && logRoll(t('m.restLongDone'), ''); } catch(_){}
}
```

> Note on `logRoll`: it may not exist — the `try/catch` keeps the
> function safe. If you prefer, drop both `logRoll` lines; they are
> purely informative.

## 6. Notes

- `_resetFeatureCard()` uses the existing `tt.avail.click` key already
  used by `resetBubbles()` at line 6084 — no new i18n needed for
  bubble tooltips.
- `restLong()` delegates spell-slot reset to `newDay()` rather than
  duplicating the loop, so the existing behaviour (reset of
  `sp-perday-*`, `sp-left-*`, `.spell-entry.expended`) is preserved.
- HP reset is guarded by `hpMax > 0` so a brand-new empty sheet does
  not auto-fill HP to `NaN`.
- Exhausted→Fatigued downgrade matches PF1e core: a night's rest
  removes exhaustion's upgrade but leaves fatigue.
- If the user has `applyCharacter()` restore condition checkboxes
  later, no change needed — we only toggle DOM state; `recalc()` at
  the end propagates the STR/DEX modifier changes.
- Auto-feature trackers without a `REST_RESET_KEY` entry (e.g. custom
  user-added features, or future classes) are treated as **long-rest
  only** for short rest and **always reset** on long rest, which is
  the safer default.

/* Check de Concentración (PF1e):
   Cuando un lanzador sufre un evento perturbador (daño, viento,
   lanzar a la defensiva, grappled, etc.) debe tirar
   1d20 + NL + mod(atributo lanzador) contra una CD variable.
   · Daño recibido: CD = 10 + daño + (nivel del conjuro si continuo)
   · Lanzar a la defensiva: CD = 15 + (2 × nivel del conjuro)
   · Terreno vigoroso / viento suave / olas: CD 10-20
   El usuario introduce la CD directamente, o un número negativo
   (p. ej. '-7') interpretado como daño ⇒ CD = 10 + 7 = 17. */
function checkConcentration() {
  const cl = parseInt(document.getElementById('caster-level')?.value) || 0;
  if(cl <= 0) { alert(t('m.concNoCL')); return; }
  const castAttr = (document.getElementById('cast-attr')||{}).value || 'int';
  // Derivamos el mod desde el atributo mostrado en la tabla de atributos.
  const attrMod = parseInt((document.getElementById(castAttr+'-mod')||{}).value) || 0;
  const input = prompt(t('m.concPrompt'), '15');
  if(input === null) return;
  const raw = String(input).trim();
  let dc;
  if(raw.startsWith('d') || raw.startsWith('D')) {
    // Formato "d7" = daño 7 → CD 10+7
    const dmg = Math.max(0, parseInt(raw.slice(1)) || 0);
    dc = 10 + dmg;
  } else {
    dc = Math.max(1, parseInt(raw) || 15);
  }
  const d20 = Math.floor(Math.random() * 20) + 1;
  const roll = d20 + cl + attrMod;
  const key = roll >= dc ? 'm.concPass' : 'm.concFail';
  const attrLabel = t('cast.'+castAttr) || castAttr.toUpperCase();
  alert(t(key, { roll, d20, cl, mod: attrMod, dc, attr: attrLabel }));
}
```

> The existing attribute-mod inputs are created as `str-mod`,
> `dex-mod`, `con-mod`, `int-mod`, `wis-mod`, `cha-mod` in the
> attributes panel. If your codebase uses a different id scheme for
> ability modifiers, substitute here — the rest of the function stays.

## 6. Notes

- Pattern deliberately mirrors `checkDispel()` (index.html line
  2008-2019) and `checkSpellResistance()` (line 2025-2035): same
  `prompt()` → validate → alert(t(key, {...})) flow, same
  no-CL guard clause.
- The CD input supports two shortcuts:
  1. **Plain number** (e.g. `15`) — used directly as DC.
  2. **`d<damage>`** (e.g. `d7`) — interpreted as 10 + damage.
  Also supports the canonical PF1e "cast defensively" case: the
  user pre-computes `15 + 2*spellLevel` and types the integer.
  (Auto-calculating that would require knowing which spell; left to
  the user.)
- Reads the ability modifier from `<cast-attr>-mod` (e.g. `int-mod`).
  The attribute panel in this codebase renders modifier inputs with
  that id convention — confirmed via `calcAttributes()` which
  populates them. If a specific id turns out different, adjust the
  single line `document.getElementById(castAttr+'-mod')`.
- Includes `{attr}` placeholder in the alert so the user sees which
  ability modifier was applied (INT / WIS / CHA as per cast-attr).
- Fails safe: if `cast-attr` is unset or the modifier cell is missing,
  `attrMod` falls back to 0 and the roll still proceeds — the user
  just sees `mod INT(0)` in the output, matching the raw d20 + CL
  formula, so no silent incorrectness.
